server/
├── GameServer
│   ├─ 클라이언트 접속 관리 (ServerSocket)
│   ├─ 각 클라이언트별 ClientHandler 스레드 생성
│   ├─ GameManager와 통신하여 게임 상태 업데이트
│
├── ClientHandler
│   ├─ 개별 클라이언트의 입력 수신 및 서버 명령 송신
│   ├─ ObjectInputStream / ObjectOutputStream 사용
│   ├─ Player 객체와 매핑
│
├── GameManager
│   ├─ 전체 게임의 턴, 이벤트, 전투 상태 관리
│   ├─ BattleSystem, MapGenerator 호출
│   ├─ 서버 기준 authoritative state 유지
│
├── BattleSystem
│   ├─ 공격, 방어, 스킬, 피해량 계산
│   ├─ 전투 결과를 GameManager에 반환
│
└── MapGenerator
    ├─ 랜덤 맵 생성 (시드 기반)
    ├─ 각 타일 타입 지정: MONSTER / SHOP / TREASURE / EVENT
    ├─ 동일 시드로 클라이언트 맵 일관성 유지



client/
├── ClientApp
│   ├─ 프로그램 진입점 (main)
│   ├─ Swing 프레임 초기화
│   ├─ GamePanel, NetworkManager 초기화
│
├── NetworkManager
│   ├─ 서버와 TCP Socket 연결
│   ├─ Serializable 객체 송수신
│   ├─ 메시지 타입별 처리 (PLAYER_MOVE, TURN_CHANGE 등)
│
├── GamePanel
│   ├─ 메인 보드 UI (맵 + 캐릭터 렌더링)
│   ├─ 사용자 입력 → NetworkManager로 전달
│   ├─ 서버에서 받은 GAME_STATE 반영
│
├── BattlePanel
│   ├─ 전투 UI (턴제 전투 버튼: 공격/방어/스킬)
│   ├─ BattleResult 수신 시 결과 출력
│
├── ShopPanel
│   ├─ 상점 UI (아이템 목록 표시 및 구매 처리)
│   ├─ 구매 요청을 서버로 전송
│
└── EventDialog
    ├─ 이벤트 발생 시 팝업 창
    ├─ 선택지에 따라 서버에 이벤트 결과 전달



shared/
├── GameState
│   ├─ 전체 게임 맵, 턴 정보, 각 Player 상태 포함
│   ├─ 서버에서 생성 → 클라이언트로 동기화
│
├── Player
│   ├─ 플레이어 기본 정보 (id, name, hp, mp, gold, inventory)
│   ├─ 현재 위치(x, y), 상태(Alive, Dead 등)
│
├── MapTile
│   ├─ 맵의 한 칸에 대한 데이터 구조
│   ├─ type (MONSTER / SHOP / TREASURE / EVENT)
│
├── Message
│   ├─ 네트워크 통신용 기본 객체 (Serializable)
│   ├─ type (PLAYER_MOVE, TURN_CHANGE, BATTLE_RESULT 등)
│   ├─ payload (Object)
│
└── Constants
    ├─ 전역 상수 (맵 크기, 턴 제한, 시드 값 등)
